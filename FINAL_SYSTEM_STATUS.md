## 🎯 System Architecture (FINAL)

### Input Pipeline
1. Raw telemetry → 30s windows
2. Uniform MinMaxScaler normalization
3. K-Means clustering (K=3) → Soft membership
4. Delta computation → Temporal signals
5. ANFIS input: **6 features** (3 soft + 3 deltas)
6. MLP surrogate training
7. Adaptation parameters output

### Final Configuration (v2.1)
- **Preprocessing**: Uniform MinMaxScaler (no feature-specific)
- **Features**: 9 active telemetry features (`timeOutOfCombat` excluded from scoring — see v2.1 change log)
- **Activity Scoring**: Per-archetype average (not sum) — equal ceiling per archetype
- **Clustering**: K=3, soft membership via inverse distance
- **ANFIS Inputs**: [soft_combat, soft_collect, soft_explore, Δcombat, Δcollect, Δexplore]
- **Target**: 1.0 - 0.1×deaths + 0.05×activity, clipped [0.5, 1.5]

### v2.1 Activity Scoring Change — Rationale

#### Problem Identified
The v2.0 Exploration score included `timeOutOfCombat` as a signal:
```
score_explore = distanceTraveled + timeSprinting + timeOutOfCombat  (sums)
```
This caused two distinct issues:

1. **Passive accumulation bias**: `timeOutOfCombat` increases any time the player is not
   in combat. On maps with sparse enemy spawns, a player who actively *seeks* combat but
   finds none accumulates a high Exploration score purely by waiting — despite their intent
   being Combat. The system would push that player toward Explorer/Collector difficulty
   settings when they wanted harder enemies, not more items or map.

2. **Redundancy with `timeInCombat`**: These two features are arithmetic complements —
   they sum to total session duration. Including both introduces inverse correlation into
   the feature space and gives the Exploration archetype a free signal boost that unfairly
   penalises combat-heavy players.

#### Fix Applied
```
score_combat  = avg(enemiesHit, damageDone, timeInCombat, kills)   — 4 features → [0,1]
score_collect = avg(itemsCollected, pickupAttempts, timeNearInteractables) — 3 → [0,1]
score_explore = avg(distanceTraveled, timeSprinting)                — 2 active → [0,1]
```

Key changes:
- **Averages instead of sums**: Each archetype now has equal max score (1.0). Previously,
  Combat had 4× the raw ceiling of a 1-feature category.
- **`timeOutOfCombat` removed**: Replaced by active-only exploration signals.

#### Impact on Training Pipeline
The centroids in `cluster_centroids.json` and the MLP weights in `anfis_mlp_weights.json`
were computed using the old formula. They must be regenerated by rerunning notebooks
04 → 05 → 06 → 07 with the updated formula in notebook 04.

**Rerun sequence:**
```
04_Activity_Contributions.ipynb  ← formula changed here
05_Clustering.ipynb              ← new centroids exported automatically
06_ANFIS_Preparation.ipynb       ← no code change needed
07_ANFIS_Training.ipynb          ← retrains on new soft membership values
```
